> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rewrite the pipeline stages — walk, filter, order, act — using the v2 node model. No hardcoded convention dir names; behavior is declared in `.dagger` files. Implement `dotd apply` and `dotd check`.

**Architecture:** Five stages in sequence: `env → walk → filter → order → act`. Walk produces raw nodes from the filesystem. Filter evaluates `@when` predicates. Order sorts via Kahn's algorithm with alpha tie-break. Act executes declared actions (link, source, no-source, compose). Each stage is a separate function/package for testability. Internal packages updated to use `internal/annotation` (new syntax), `internal/dagger` (new format), `internal/node` (logical names).

**Tech Stack:** Go, existing `internal/predicate/` (unchanged), `gopkg.in/yaml.v3`.

**Prerequisites:** Plan 1 (Foundation) and Plan 2 (Env & Config) complete.

## Context

### What changes from v1

| v1 | v2 |
|----|-----|
| Convention dirs hardcoded: `shellrc/`, `conf/`, `bin/` | No hardcoded names — `.dagger` defaults declare behavior |
| `Kind` enum (KindScript/KindConf/KindBin) drives behavior | Actions list drives behavior (source, link, no-source) |
| `.dotd.yaml` loaded by walk | `.dagger` loaded by walk |
| Annotation `Value` field | Annotation `Args` field |
| `@symlink dest` | `@link(dest)` |
| `@when os=macos` | `@when(os=macos)` |

### v2 node produced by walk

```go
// RawNode is a file discovered during walk, before predicate evaluation.
type RawNode struct {
    Path          string              // absolute filesystem path
    LogicalName   string              // derived via node.DeriveName
    EffectiveWhen string              // merged from file @when + ancestor defaults
    Actions       []Action            // merged from file annotations + ancestor defaults
    LinkRoot      string              // from nearest ancestor .dagger with link_root set
    IsCompose     bool                // true if inside a composition-enabled dir
    ComposeTarget string              // abs path to compose target dir (if IsCompose)
}

type Action struct {
    Type string // "link", "source", "no-source", "compose"
    Dest string // only for "link"
}
```

### Action merging rules

For each file, effective actions = ancestor `defaults.actions` (outermost first) overridden by file's own annotations/`.dagger` actions. An explicit action of the same type replaces the default; a different type adds to it.

## Task 1: Walk Stage

Walk traverses the dotfiles repo root. For each file:
1. Load `.dagger` files from all ancestor directories, accumulate defaults cascade
2. Parse file annotations (new `@key(args)` syntax)
3. Compute logical name via `node.DeriveName`
4. Compute effective when (file's `@when` ANDed with cascaded defaults)
5. Compute effective actions (defaults + file overrides per action-type merging rules)
6. Compute link_root from nearest ancestor with `link_root` set

Skip directories with a `.dagger` that has `composition: enabled: true` — their children are compose fragments (different node type).

Skip `.dagger` files themselves — they're metadata, not dotfiles.

- [ ] **Step 1.1: Write walk tests against `cmd/dotd/testdata/dotfiles/`**

Tests verify that a known fixture directory produces the expected `[]RawNode` list. Spot-check: logical names, effective when, actions from defaults.

Key test cases:
- File in a directory with `defaults.actions: [source]` → action list includes `source`
- File with `@link(~/.tmux.conf)` → actions includes `{Type:"link", Dest:"~/.tmux.conf"}`
- File with `@when(context=work)` inside dir with `defaults.when: os=macos` → EffectiveWhen = `(os=macos) AND (context=work)`
- `.dagger` file itself → not in output

- [ ] **Step 1.2: Run tests to verify they fail**

```bash

```

- [ ] **Step 1.3: Implement `internal/pipeline/walk.go`**

Key implementation notes:
- Use `filepath.WalkDir` with a state struct that tracks the `.dagger` cascade per directory
- At each dir entry: load `.dagger` for that dir, push defaults onto cascade stack
- At each file entry: merge cascade, parse annotations, build `RawNode`
- Compose target dirs: flag children as `IsCompose: true`, do not recurse into sub-compose targets (error)

## Task 3: Order Stage

Compose targets use isolated per-target sub-DAG: fragments are sorted among themselves, not globally.

## Task 4: Act Stage

Execute actions for each node in DAG order. Four action types:

- **`source`** — register node for init.sh generation
- **`no-source`** — suppress sourcing (even if defaults include source)
- **`link(dest)`** — create symlink at dest → node path; expand `~/`; conflict detection
- **`compose`** — concat active fragments in sub-DAG order → write generated file; register synthetic node for subsequent actions

Compose output path: `{generated_dir}/{output-logical-name}` (from node's logical name or `.dagger` `name`).

- [ ] **Step 4.1: Write tests**

Tests use temp directories. Verify:
- `source` action → node appears in sourced list
- `no-source` suppresses sourcing even with source default
- `link(~/.tmux.conf)` → symlink created at correct expanded path
- `link` conflict (two nodes → same dest) → error
- `compose` → generated file contains concatenated fragment contents in order

- [ ] **Step 4.2: Implement `internal/pipeline/act.go`**

```go
type ActResult struct {
    Sourced  []RawNode // nodes to be sourced in init.sh, in order
    Links    []Link    // symlinks to create
    Generated []Generated // compose output files
}

}

    Content []byte
}

// Act executes actions for ordered nodes. Returns results for init.sh generation
// and linking. DryRun: validate without writing.
```
