## Issues found + fixes (all applied to plan + spec)

| # | Sev | Problem | Fix |
|---|-----|---------|-----|
| 1 | HIGH | `TestAnnotate_NonTTY` asserted on captured `out`, but root sets `SilenceErrors: true` (main.go:63) → message is on `err`, not the buffer. Test would always fail. | Assert on `err.Error()`, discard `out`. Mirrors existing runMayFail tests (integration_test.go:362). |
| 2 | HIGH | Design said "empty input → remove" for KindText, but `promptDefault` returns the default on empty input, so an existing single value was **unremovable** (Enter re-affirmed it). | KindText → `huh.NewInput` pre-filled with existing value; clearing the field (empty) removes. |
| 3 | MED | `Write` stripped `^(#\|//)\s*@\w+` **anywhere** in the file, but `Scan` stops at the first code line. A body comment like `# @param` would be silently deleted. | Bounded strip + insertion to the **header block** (shebang + contiguous comment/blank lines, matching `Scan`). Added `TestWrite_BodyAnnotationLikeCommentPreserved`. |
| 4 | MED | Menu used `huh.Select` (raw-mode) while value prompts used a `bufio.Reader` over `cmd.InOrStdin()` — mixing raw + line-buffered reads on one stdin can drop typed-ahead bytes. Footgun matches the open `.claude/TODO.md` huh/cobra question. | Unified **all** wizard input on `huh` (Select/Input/Confirm); removed the bufio reader. Added a manual-TTY smoke-test item to the Self-Review Checklist (huh prompts aren't unit-testable). |
| 5 | LOW | `@name` is semantically single-valued but is `KindText` (appendable). | Documented as accepted Phase-1 behavior (multiple entries only from pre-existing input; collapse on next edit). No interface surface added. |
| 6 | LOW | (a) stripping all annotations can leave an orphan blank line; (b) `append(preserved, lines...)` could alias the caller's backing array. | (a) noted as cosmetic; (b) `Write` now copies into a fresh slice. |

Bonus from #4: KindBool upgraded `promptYN` → `huh.NewConfirm` (Affirmative "Set" / Negative
"Remove"), pre-populated with current state.
