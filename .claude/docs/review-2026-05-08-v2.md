## CRITICAL — blocks merge

### C-1: Integration fixtures use v1 syntax throughout
`cmd/dotd/testdata/dotfiles/` — entire directory

- `.dotd.yaml` files instead of `.dagger`
- Annotation format `@when os=linux` instead of `@when(os=linux)` — silently inert in v2 scanner
- Nested `env.yaml` (`env:\n  context: personal`) — v2 `map[string]string` decoder crashes on this
- No `.dagger` files for `conf/`/`shellrc/` dirs — no link or source actions applied

All integration tests guarded by `//go:build integration`; pass in `go test ./...` but fail completely when run.

**Fix:** Rewrite entire fixture tree for v2: `.dagger` files with v2 schema, `@when(...)` annotation syntax, flat `env.yaml`.

### C-2: Duplicate `TestCheckAfterApply` — compile error under `-tags integration`
`cmd/dotd/main_test.go:199` and `cmd/dotd/integration_test.go:277`

`go test -tags integration ./cmd/dotd/` fails to compile with redeclaration error.

**Fix:** Rename unit test to `TestCheckAfterApply_Unit`.

### C-3: `@name` annotation parsed but never applied to `LogicalName`
`internal/pipeline/walk.go:108`

`node.DeriveName(rel)` always used. `@name(override)` is a no-op. Breaks the variant pattern (two files, same `@name`, mutually exclusive `@when`).

**Fix:** After scanning annotations, check for `name` key; if present use its args as `LogicalName`.

### C-4: `.dagger` `files:` dict parsed but never emitted as nodes
`internal/pipeline/walk.go` — Walk function

`cfg.Files` (a `map[string]NamedNode`) loaded into `daggerMap`, never iterated. Files listed there get no `when`/`name`/`actions` applied and never appear in the pipeline. The entire "configure non-annotatable files" feature is a no-op.

**Fix:** After filesystem walk, iterate each dir's `cfg.Files`; create `RawNode` entries applying each `NamedNode`'s fields.

### C-5: `ecosystem.ConfigFile` still `.dotd.yaml`; `setup.go` emits v1 schema
`internal/ecosystem/ecosystem.go:15`, `cmd/dotd/setup.go:142`

`daggerYAML()` generates `dotd: { when:, defaults: }` v1 YAML; v2 parser uses `KnownFields(true)` and would reject it.

**Fix:** Change `ConfigFile = ".dagger"`; rewrite `daggerYAML()` to emit valid v2 schema.
