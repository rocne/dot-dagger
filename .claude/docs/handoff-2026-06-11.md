## Patterns introduced

### `pathFlagOwners` + `hideIrrelevantInheritedFlags` (`cmd/dotd/main.go`)

Persistent path/mutation flags (`--init-file`, `--link-root`, `--bin-dir`,
`--generated-dir`, `--dry-run`, `--force`) used to appear under "Global
Flags" on every subcommand's `--help`, including commands that don't act
on them (`dotd config get`, `dotd env show`, …). `pathFlagOwners` is a
flag → command-path allowlist. The root `SetHelpFunc` toggles `Hidden`
on inherited flags before rendering and a deferred closure restores prior
state — so subsequent `--help` invocations in the same process (notably
in tests) are unaffected. Flags still parse if passed where unlisted;
they're only hidden from help. Root `--help` (and `dotd help`) skip
filtering and remain the canonical flag reference.

If you add a new persistent flag that's only meaningful on some commands,
add an entry to `pathFlagOwners`.

### `usageError` + exit-code dichotomy (`cmd/dotd/errors.go`)

```go
type usageError struct{ err error }
```

`main()` now:

```
errors.Is(err, errUserAborted) → "cancelled", exit 1
errors.As(err, &usageError)    → "error:" + optional "hint:", exit 2
otherwise                      → "error:" + optional "hint:", exit 1
```

Wired via `root.SetFlagErrorFunc` (flag parse failures) and a
`usageArgs(cobra.PositionalArgs)` wrapper for the bare `cobra.ExactArgs`
call sites (`adopt`, `annotate`, `bundle`, `completion`). The two
hand-rolled validators `envKeyArgs` / `configKeyArgs` wrap their
`hintError` in `usageError`. `usageError.Unwrap()` returns the inner so
`hintError`'s `Hint()` still surfaces.

Heuristic for future work: did cobra's arg-shape / flag-parse layer
reject the invocation? → usage (exit 2). Did the business logic fail
(unknown config key, missing env key, walk error, etc.)? → runtime (exit 1).

### `env show` text format

Text output is now uniform `key=value`, one per line. The previous
inline source column (`\t[$(…)]`) is gone — invisible whitespace gave
piped consumers an unpredictable column count. The expression survives
in `env show --json` as a per-entry `expression` field; the `Long`
help and e2e tests cover both shapes.

### `runWithStdin(t, io.Reader, args...)` test helper

`cmd/dotd/main_test.go:14`. New helper used by every interactive-prompt
test (setup, teardown cancel, unapply cancel). The old `run(t, args...)`
is now a one-liner that forwards `nil` stdin. Removed ~50 lines of
re-wired cobra plumbing across eight tests. `runErr` (next to it) is the
return-only variant used by the exit-code tests — kept separate because
it returns `error`, not `(string, error)`.
