## Patterns introduced

### `Hinter` interface + `hintError` (`cmd/dotd/errors.go`)

Errors that want a `hint:` line on their own implement `Hinter`. Root error
handler in `main.go` does `errors.As(err, &h)` and renders:

```
error: <message>
hint:  <recommended action>
```

Used by `annotateKeyError` (missing predicate key) and `dotd init`'s
"no config found", plus `config get`/`set` and `env get`/`set` custom
arg validators. Any new error path that has a clear next-step should
return a `hintError` instead of embedding `\nhint:` in the message.

Exception: `internal/annotation/registry.go:52` (`@when` validate) still
embeds `\nhint:` in the message — that error is rendered inline by
`huh` form validation during `dotd annotate`, so the embedded form is
correct in that user-facing context.

### `--json` shape

All `--json` outputs are array-of-objects with snake_case JSON tags, so
fields can be added later without breaking consumers. See `cmd/dotd/*_cmd.go`
for shapes.

### Cobra help template fix

`root.SetHelpFunc` was calling `cmd.Usage()` which uses `UsageTemplate` and
omits `Long`. Now it prints `Long` (or `Short`) before `Usage`, matching
Cobra's default Help. If anyone adds a new command with a `Long`, it will
render automatically.

### `Reference:` command group

Root help now has four groups: Core, Configuration, Advanced, Reference.
`concepts` and `completion` live in Reference. New non-action commands
that exist to document things should join Reference.
