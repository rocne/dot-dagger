## Conventions established/reaffirmed this session

- **Channel policy** (new, in audit-guide): mutation results → stdout,
  never --quiet-gated; cfg.log diagnostics only.
- **ToolD rule** (new): prose/help may say `dotd`; generated/executed
  content must use `ecosystem.ToolD`/`ecosystem.Name`.
- **Dry-run = skip writes only** — the dry-run plan must equal the real plan.
- **Wrapped-when invariant**: every when-source paren-wrapped before AND-join.
- Single owners now exist for: atomic writes (fileutil.WriteAtomic), shell
  quoting (fileutil.ShellQuote), generated headers, predicate syntax help,
  pluralization, --json flag, key-arg validation, pipeline preamble
  (walkActive), bin prefix. **New code must route through them.**

## Lessons (also in audit-guide + memory)

- **Tests/e2e encoded the bugs**: compose.sh asserted exit-0-on-stale,
  TestAct_Compose_DryRun asserted empty dry-run content, adopt.sh relied on
  non-TTY auto-accept. When fixing exit codes/prompts/dry-run, grep
  `*_test.go` AND `test/e2e/*.sh` for the old behavior first.
- e2e runner stops at first failing script — one red script masks the rest;
  sweep all scripts for affected commands preemptively.
- Advertised-vs-honored flags: cross-check `pathFlagOwners` against RunE
  bodies; that's how B1 was found.
- Branch switches invalidate Read state — re-read before editing (hit
  several times; harness enforces).
