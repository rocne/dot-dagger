## Key facts about the codebase

- Config file: `.dagger` (flat YAML). `ecosystem.ConfigFile = ".dagger"`, `LegacyConfigFile = ".dotd.yaml"`
- Convention dirs: `shellrc/` (source), `conf/` (link to ~), `bin/` (link to bindir). Created by `dotd setup`. Each gets a `.dagger` with `defaults.actions: [source]` or `link_root + defaults.actions: [link]`
- Action annotations: `@action source|no-source|link(<dest>)|link|compose`. Aliases: `@source`, `@no-source`, `@link(<dest>)`, `@symlink` (legacy)
- Compose: directory with `compose: true` in `.dagger` → fragments concatenated → generated file. `dotd compose list/check`
- `dotr` and `dotd-new` in repo root are untracked build artifacts, not documented, not public features
