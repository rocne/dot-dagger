## Key decisions

- **Both layers**: each gap gets shell e2e (real binary in Docker) + Go integration test
- **No expect**: `promptConfirm`/`promptYN`/`promptDefault` have no TTY check — piped stdin works everywhere. Only `adopt` uses `isTTYStdin()`, and it auto-skips prompt in non-TTY mode. Expect infra dropped.
- **Teardown ignores `--config`**: always calls `dotcfg.DefaultPath()` directly. Tests use `XDG_CONFIG_HOME=/tmp/xdg` to control target path.
- **`dagger.LoadFile` returns empty config on ENOENT** — adopt works fine without a root `.dagger` in either fixture.
- **`init.sh` pipe**: `y\n\ny\n\ny\n\nn\n` (yes to all 3 dirs + explicit n to source-line prompt)
- **`setup.sh`**: set `DOTFILES=/fixture` to pin default path for deterministic assertion
- **Fixtures stay separate**: e2e fixture (`test/e2e/fixture/`) and integration fixture (`cmd/dotd/testdata/dotfiles/`) remain independent
