Two stacks today:
- **huh** (`prompts.go`): `promptMenu/Select/Input/Bool/Inputs` — used by annotate, adopt, filter_prompt.
- **bufio** (`prompts.go` lower half): `promptConfirm`, `promptDefault`, `promptYN`, `promptPath`, `printField/fieldPrompt` — used by setup, init, teardown, unapply.

I probed huh v1.0.0 accessible mode (the non-TTY path) to see if it can replace
the bufio prompts without behavior loss. **It can:**
- Confirm respects the initial value as the Enter-default (`[Y/n]` vs `[y/N]`).
- Confirm on EOF → `false` (matches the "never auto-accept on closed stdin" rule).
- Input on empty line / EOF → returns the prefill (matches `promptDefault`).

**Recommendation: standardize on huh.** `promptConfirm/Default/YN/Path` become
thin huh wrappers. Why deferred: it rewrites the interactive flow of 4 commands
AND their tests — setup/init tests pipe `\n`-delimited lines through `byteReader`;
each prompt's driving changes. That's a focused PR of its own, not a tail-end
change. The handoff before this one already flagged D6 as "spec-level — brainstorm
first"; the probe removes the technical unknown, so it's now a straight (if large)
refactor whenever picked up.
