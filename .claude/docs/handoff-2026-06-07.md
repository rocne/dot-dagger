## Key design decisions (non-obvious)

- `AnnotationType` is pure domain — no `io.Reader`/`io.Writer`; all I/O in command layer
- Staged list is `[]stagedEntry` (ordered slice), not map — supports multi-valued annotations (`@when` × N)
- KindBool: set-one-or-remove-all (not multi-valued)
- KindText/KindChoice: replace-if-one-existing, append-if-zero-or-multiple
- Unknown annotation keys → `preserved []string` (written through unchanged, never dropped)
- Writer always writes on Done — empty staged = strip all known annotations (intentional)
- `promptMenu` uses `huh.Select` (accepted exception to cobra I/O routing, same as `adopt.go`)
- `isTTYStdin()` — no args, checks `os.Stdin.Fd()` directly (in `filter_prompt.go`)
- `@action` options: `source`, `no-source`, `link` (matches `ActionSource`/`ActionNoSource`/`ActionLink` constants in `walk.go`)
- File must be regular + inside `cfg.files` (validated with `filepath.Rel` + `strings.HasPrefix(rel, "..")`)
- `annotate` registered in core group alongside `adopt`, `apply`, etc.
